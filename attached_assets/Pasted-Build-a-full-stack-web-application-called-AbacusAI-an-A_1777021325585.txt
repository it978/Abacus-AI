Build a full-stack web application called "AbacusAI" — an AI-powered abacus 
learning platform with Free + Paid subscription tiers targeting learners 
from age 5 to adults (Kids, Tweens, Teens, Adults).

=============================================================
TECH STACK
=============================================================
- Frontend: React + Vite + Tailwind CSS
- Backend: Node.js + Express
- Database: PostgreSQL (via Prisma ORM)
- Auth: JWT-based (email/password) + Google OAuth
- Payments: Razorpay (Free tier + Paid subscription)
- AI: Anthropic Claude API (claude-sonnet-4-20250514)
- Hosting-ready: all env vars in .env file

=============================================================
USER ROLES
=============================================================
1. Student (child or adult learner)
2. Parent (manages child accounts, pays subscription)
3. Teacher (manages a class of students)
4. Admin (full dashboard, user management, revenue view)

=============================================================
AI FEATURES (Claude API Integration)
=============================================================

1. AI TUTOR / HINTS:
   After 2 wrong answers, AI explains concept in age-appropriate language.
   
   API call:
   POST /api/v1/ai/hint
   Body: { studentAge, level, problem, wrongAnswers }
   
   System prompt:
   "You are a friendly, encouraging abacus tutor for a [age]-year-old student 
   at Level [X]. They got this problem wrong twice: [problem]. Their wrong 
   answers were: [wrongAnswers]. Explain the correct method in simple, 
   age-appropriate language in under 80 words. Be warm and encouraging."
   
   Response: { hint: "string", encouragement: "string" }

2. AI-GENERATED PRACTICE PROBLEMS:
   Generate 10 fresh problems per session based on level + past performance.
   
   API call:
   POST /api/v1/ai/generate-problems
   Body: { level, recentAccuracy, focusArea, studentAge }
   
   System prompt:
   "Generate exactly 10 abacus math problems for a Level [X] student aged [age].
   Their recent accuracy is [X]%. Focus area: [focusArea].
   Return ONLY a JSON array, no markdown, no explanation:
   [{ question, answer, hint, difficulty: 'easy|medium|hard' }]"
   
   Parse response as JSON array directly.

3. AI PROGRESS TRACKING & REPORTS:
   Weekly narrative report generated by Claude from session data.
   
   API call:
   POST /api/v1/ai/weekly-report
   Body: { studentName, age, sessionsThisWeek, avgScore, 
           levelProgress, weakAreas, strongAreas }
   
   System prompt:
   "You are an abacus learning coach. Write a friendly, encouraging weekly 
   progress report for [studentName] aged [age]. Data: [sessionData].
   Include: what improved, what needs work, a specific practice tip, 
   and a motivational close. Format as HTML with <h3>, <p>, <ul> tags. 
   Keep it under 200 words."
   
   Save as HTML → convert to PDF → email to parent via SMTP.

4. AI VOICE ASSISTANT (PAID only):
   Uses Web Speech API + Claude for intent processing.
   
   Recognized voice commands:
   - "Next problem" → load next question
   - "Give me a hint" → trigger AI hint
   - "Read the question" → SpeechSynthesis reads aloud
   - "Show answer" → reveal answer (counts as wrong attempt)
   - "How many beads?" → AI explains bead position
   
   Flow: SpeechRecognition → send transcript to Claude →
   Claude returns { intent, response_text, action } →
   Frontend executes action + speaks response_text via SpeechSynthesis

5. AI ADAPTIVE DIFFICULTY:
   After each session, send score to Claude for next session calibration.
   
   API call:
   POST /api/v1/ai/adapt
   Body: { sessionScore, timePerProblem, hintsUsed, 
           currentLevel, recentHistory }
   
   System prompt:
   "You are an adaptive learning engine. Based on this student's session data,
   return ONLY JSON (no markdown):
   { next_difficulty: 'easy|medium|hard', 
     recommended_problem_count: number,
     focus_area: 'string',
     should_advance_level: boolean,
     coach_tip: 'string under 20 words' }"

ANIMATED ABACUS COMPONENT (React SVG)
=============================================================
Build a reusable <AbacusBoard /> React component:
- 9 vertical rods
- Each rod: 1 heaven bead (top, separated by beam) + 4 earth beads
- Heaven bead = value 5, Earth bead = value 1
- Click OR drag to move beads toward/away from beam
- CSS transition: 0.15s ease snap animation on bead movement
- Props:
  - mode: 'display' | 'interactive' | 'flash'
  - value: number (display mode auto-positions beads)
  - onValueChange: callback (interactive mode)
  - theme: 'kids' | 'teen' | 'adult' | 'dark'
- Flash mode: show number for 1.5s then hide beads (mental math)
- Themes:
  - kids:  amber beads, coral frame, sky blue background
  - teen:  indigo beads, teal accents, white background  
  - adult: slate beads, gold accents, dark background
  - dark:  neon green beads, charcoal frame

DATABASE SCHEMA (Prisma)
=============================================================

model User {
  id             String   @id @default(uuid())
  name           String
  email          String   @unique
  password       String?
  googleId       String?
  role           Role     @default(STUDENT)
  age            Int?
  avatar         String?
  createdAt      DateTime @default(now())
  subscription   Subscription?
  student        Student?
  children       Student[] @relation("ParentChildren")
}

model Student {
  id          String    @id @default(uuid())
  userId      String    @unique
  user        User      @relation(fields: [userId], references: [id])
  level       Int       @default(1)
  xp          Int       @default(0)
  streak      Int       @default(0)
  lastActive  DateTime  @default(now())
  parentId    String?
  parent      User?     @relation("ParentChildren", fields: [parentId], references: [id])
  sessions    Session[]
  reports     Report[]
}

model Session {
  id          String   @id @default(uuid())
  studentId   String
  student     Student  @relation(fields: [studentId], references: [id])
  level       Int
  score       Int
  accuracy    Float
  duration    Int      // seconds
  problems    Json     // array of {question, answer, studentAnswer, correct}
  hintsUsed   Int      @default(0)
  aiAdapt     Json?    // adaptation output from Claude
  createdAt   DateTime @default(now())
}

model Subscription {
  id            String   @id @default(uuid())
  userId        String   @unique
  user          User     @relation(fields: [userId], references: [id])
  plan          Plan     @default(FREE)
  status        Status   @default(ACTIVE)
  razorpaySubId String?
  startDate     DateTime @default(now())
  endDate       DateTime?
  updatedAt     DateTime @updatedAt
}

model Report {
  id          String   @id @default(uuid())
  studentId   String
  student     Student  @relation(fields: [studentId], references: [id])
  htmlContent String
  pdfUrl      String?
  weekOf      DateTime
  sentAt      DateTime?
  createdAt   DateTime @default(now())
}

enum Role   { STUDENT PARENT TEACHER ADMIN }
enum Plan   { FREE STAR SCHOOL }
enum Status { ACTIVE EXPIRED CANCELLED }
